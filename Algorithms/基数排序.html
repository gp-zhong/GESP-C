算法基础

基数排序

一种非比较型整数排序算法，通过按位排序实现高效的数据排序

算法简介

基数排序（Radix Sort）是一种非比较型整数排序算法，其基本思想是：

  • 按照数字的位数进行排序，从最低位（个位）开始，逐位向最高位排序
  • 对每一位上的数字排序采用稳定的排序算法，如计数排序
  • 通过多轮按位排序，最终实现整个数组的排序

基数排序是一种稳定的排序算法，避开了比较操作，适用于整数或可转换为整数的数据，如字符串排序。其时间复杂度为 O(nk)，其中 n 是数组长度，k 是数字位数。

算法特点

优点

  • 对于特定范围的整数排序速度快，避免了元素比较操作
  • 是一种稳定的排序算法，保持相同元素的相对顺序
  • 适合排序位数少的数字和字符串
  • 可以轻松处理负数和小数（通过预处理）

缺点

  • 需要额外的空间用于存储临时数组
  • 当数字位数较多时，排序效率可能下降
  • 对于不同位数的数字，需要额外的处理
  • 相比比较型排序，实现相对复杂些

算法原理

基数排序的基本原理是按照数字的位数进行分类，具体步骤如下：

  1. 确定最大位数：找出数组中最大的数，并确定其位数
  2. 按位排序：从最低位（个位）开始，对每一位上的数字进行排序
  3. 收集结果：按照排序后的顺序收集元素，形成新的数组
  4. 重复执行：对更高位重复步骤2和3，直到对最高位排序完成

实现方式

基数排序有两种主要的实现方式：

  • LSD（Least Significant Digit）：从最低位开始排序，适合位数相同的数值
  • MSD（Most Significant Digit）：从最高位开始排序，适合位数不同的数值

常见实现中，基数排序每一位上的排序通常使用计数排序或桶排序，因为它们对于有限范围的整数（0-9）排序非常高效。

LSD 基数排序过程总结

  1. 第一轮：按个位排序，相当于将数字分到0-9的十个桶中，并按桶的顺序收集
  2. 第二轮：按十位排序，将上一轮结果再次分到0-9的桶中并收集
  3. 第三轮：按百位排序，重复上述过程
  4. 排序完成后，数组已按照数值大小排好序

每一轮排序中，必须使用稳定的排序算法（如计数排序）才能保证基数排序的正确性。

代码实现

C++ 实现

// 基数排序 - C++ 实现
#include <iostream>
#include <vector>
#include <algorithm>
#include <cmath>

// 获取数字在指定位上的数值（个位、十位、百位...）
int getDigit(int num, int place) {
    return (num / static_cast<int>(std::pow(10, place))) % 10;
}

// 计数排序的变种，用于按数字的某一位进行排序
void countingSortByDigit(std::vector<int>& arr, int place) {
    const int n = arr.size();
    if (n <= 1) return;
    
    std::vector<int> output(n);
    std::vector<int> count(10, 0); // 0-9 共十个数字
    
    // 统计每个数字在当前位上出现的次数
    for (int i = 0; i < n; i++) {
        int digit = getDigit(arr[i], place);
        count[digit]++;
    }
    
    // 计算累加计数，确定位置
    for (int i = 1; i < 10; i++) {
        count[i] += count[i - 1];
    }
    
    // 从后向前遍历，保持稳定性
    for (int i = n - 1; i >= 0; i--) {
        int digit = getDigit(arr[i], place);
        output[count[digit] - 1] = arr[i];
        count[digit]--;
    }
    
    // 将排序结果复制回原数组
    for (int i = 0; i < n; i++) {
        arr[i] = output[i];
    }
}

// 基数排序主函数
void radixSort(std::vector<int>& arr) {
    // 找出最大值
    int max_val = *std::max_element(arr.begin(), arr.end());
    
    // 计算最大值的位数
    int max_digits = 0;
    while (max_val > 0) {
        max_digits++;
        max_val /= 10;
    }
    
    // 从个位开始，对每一位进行排序
    for (int place = 0; place < max_digits; place++) {
        countingSortByDigit(arr, place);
    }
}

// 示例使用
int main() {
    std::vector<int> arr = {170, 45, 75, 90, 802, 24, 2, 66};
    
    std::cout << "排序前: ";
    for (int num : arr) {
        std::cout << num << " ";
    }
    std::cout << std::endl;
    
    radixSort(arr);
    
    std::cout << "排序后: ";
    for (int num : arr) {
        std::cout << num << " ";
    }
    std::cout << std::endl;
    
    return 0;
}

优化技巧

  • 内置序列位数处理：实现时可以检测数列内元素的最大位数，只排序到必要的位数
  • 负数处理：对于包含负数的数组，可以先将所有数加上一个偏移量使其变为非负数，排序后再减去偏移量
  • 桶排序替代：在某些情况下，使用桶排序代替计数排序可能更高效
  • 位操作优化：使用位运算可以更快地提取数字的特定位
  • MSD优化：对于位数差异大的数据集，可以考虑使用MSD（最高位优先）基数排序

时间复杂度分析

最佳情况

O(nk)

其中 n 是数组长度，k 是最大数字的位数

平均情况

O(nk)

基数排序的时间复杂度不受输入数据的分布影响

最差情况

O(nk)

即使在最差情况下，时间复杂度仍然是 O(nk)

空间复杂度

基数排序的空间复杂度主要来自于临时数组和桶空间：

  • 输出数组：O(n) - 存储每次按位排序的结果
  • 计数数组：O(k) - 对于十进制数，每位上的数字范围是0-9，需要10个桶
  • 总空间复杂度：O(n+k)，通常k=10（十进制数的位数范围）

基数排序与比较排序的比较

基数排序是一种非比较排序算法，它跳过了元素间的直接比较，而是依赖于特定的数据结构：

  • 基于比较的排序算法（如快速排序、归并排序）的理论下界是 O(n log n)
  • 基数排序的时间复杂度为 O(nk)，当 k 远小于 log n 时，基数排序可以更高效
  • 对于固定范围内的整数（如32位整数），k 是常数，此时基数排序的时间复杂度可以视为线性 O(n)
  • 基数排序是稳定的，相同元素的相对顺序在排序后保持不变

性能影响因素

基数排序的性能受以下因素影响：

  • 数据位数 (k)：位数多的数据排序需要更多轮次，影响总体性能
  • 基数大小：十进制数使用10个桶，二进制使用2个桶，基数越大需要的桶越多但轮次越少
  • 数据分布：位数差异大的数据集用MSD(Most Significant Digit)方式可能更高效
  • 实现方式：每位排序使用的算法（计数排序或桶排序）会影响整体效率

应用场景

基数排序在特定场景下表现优异，常用于以下应用：

整数排序

对于固定范围的整数排序（如32位或64位整数），基数排序可以实现线性时间复杂度

字符串排序

对字符串按字典序排序，特别是长度相近的字符串，基数排序表现很好

IP地址排序

网络中的IP地址排序，可以视为对固定格式数据的排序

日期时间排序

对格式化的日期和时间进行排序，按年、月、日、时、分、秒的顺序

经典应用案例

1. 后缀数组构建

后缀数组是字符串处理中的重要数据结构，基数排序可用于高效构建后缀数组

利用基数排序的思想对字符串的后缀进行排序，构建出高效的后缀数组结构

2. 文件系统中的大文件排序

处理大型日志文件或数据库记录时，基数排序可以作为外部排序的一部分

将大文件分割处理，每次处理一个数字位，可以在有限内存中处理超大数据集

3. 电话号码排序

大型电话簿或通讯录中的号码排序

电话号码通常是固定长度的数字序列，非常适合使用基数排序进行处理
