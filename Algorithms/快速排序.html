算法基础

快速排序

一种高效的分治排序算法，通过基准元素划分数组实现快速排序

算法简介

快速排序（Quick Sort）是一种分治策略的排序算法，由计算机科学家 Tony Hoare 于1960年提出。它的基本思想是：

  • 通过选择一个"基准"（pivot）元素，将数组分为两部分
  • 所有小于基准的元素移到基准前面，所有大于基准的元素移到基准后面
  • 递归地对分割后的两个子数组进行相同操作

快速排序是一种不稳定的排序算法，但在实际应用中通常比其他排序算法（如归并排序、堆排序）更高效，平均时间复杂度为 O(n log n)。

算法特点

优点

  • 平均时间复杂度 O(n log n)，实际运行效率通常很高
  • 原地排序算法，不需要额外的存储空间
  • 分治思想，可以很好地利用缓存局部性原理
  • 适合处理大规模数据

缺点

  • 最坏情况下时间复杂度为 O(n²)
  • 对于已经排序或基本排序的数组，性能下降明显
  • 不稳定排序，相同元素的相对位置可能改变
  • 递归实现可能导致栈溢出（深度最大可达 O(n)）

算法原理

快速排序基于分治法，其核心步骤如下：

  1. 选择基准：从数组中选择一个元素作为基准（pivot）
  2. 分区操作：重新排列数组，所有小于基准的元素放在基准前面，所有大于基准的元素放在基准后面。在分区结束后，基准处于最终位置
  3. 递归排序：递归地将小于基准的子数组和大于基准的子数组进行排序

基准选择策略

基准的选择对快速排序的性能有显著影响：

  • 固定位置：选择第一个、最后一个或中间的元素（简单但在部分有序数据上表现差）
  • 随机选择：随机选择一个元素作为基准（可以有效避免最坏情况）
  • 三数取中：取首、尾、中间三个元素的中位数作为基准（平衡效率和简单性）

代码实现

C++ 实现

// 快速排序 - C++ 实现
#include <iostream>
#include <vector>

// 分区函数
int partition(std::vector<int>& arr, int low, int high) {
    // 选择最右边的元素作为基准
    int pivot = arr[high];
    
    // 初始化小于基准元素的区域指针
    int i = low - 1;
    
    // 遍历数组，将小于基准的元素放到左边
    for (int j = low; j < high; j++) {
        if (arr[j] <= pivot) {
            i++;
            std::swap(arr[i], arr[j]);
        }
    }
    
    // 将基准元素放到正确的位置
    std::swap(arr[i + 1], arr[high]);
    return i + 1;
}

// 快速排序函数
void quickSort(std::vector<int>& arr, int low, int high) {
    if (low < high) {
        // 获取分区位置
        int pi = partition(arr, low, high);
        
        // 递归排序基准左边的元素
        quickSort(arr, low, pi - 1);
        
        // 递归排序基准右边的元素
        quickSort(arr, pi + 1, high);
    }
}

// 快速排序函数封装
void quickSort(std::vector<int>& arr) {
    if (arr.size() > 1) {
        quickSort(arr, 0, arr.size() - 1);
    }
}

// 示例使用
int main() {
    std::vector<int> arr = {7, 2, 1, 6, 8, 5, 3, 4};
    
    std::cout << "排序前: ";
    for (int num : arr) {
        std::cout << num << " ";
    }
    std::cout << std::endl;
    
    quickSort(arr);
    
    std::cout << "排序后: ";
    for (int num : arr) {
        std::cout << num << " ";
    }
    std::cout << std::endl;
    
    return 0;
}

优化技巧

  • 三数取中法：选择左端、右端和中心位置上的三个元素，取它们的中位数作为枢纽元素，可以避免对已排序数组的最坏情况
  • 随机选择基准：随机选择一个元素作为基准，可以有效避免最坏情况
  • 处理小规模数组：对于小规模数组（如长度小于10），使用插入排序更高效
  • 尾递归优化：通过循环代替对较大子数组的递归调用，可以减少栈空间的使用
  • 双路快排：将等于基准的元素均匀分布在两侧，处理大量重复元素时效率更高

时间复杂度分析

最佳情况

O(n log n)

每次分区都能将数组分成大小相近的两部分

平均情况

O(n log n)

大多数情况下，快速排序都能表现出理想的时间复杂度

最差情况

O(n²)

当数组已基本有序或完全有序时，每次只能排定一个元素

空间复杂度

快速排序的空间复杂度主要取决于递归调用栈的深度：

  • 最佳情况：O(log n) - 每次均匀分区，递归树高度为 log n
  • 最差情况：O(n) - 当数组已排序或逆序时，递归深度可能达到 n

应用场景

快速排序是最常用的高效排序算法之一，适用于多种场景：

系统实现

许多编程语言的标准库排序函数使用快速排序或其变体，如C++的std::sort、Java的Arrays.sort等

大数据处理

对大规模数据集进行排序时，快速排序通常比其他O(n log n)算法表现更好

数据库系统

数据库系统中的排序操作经常使用快速排序或其变体

算法设计

快速排序的分区思想被广泛应用于其他算法中，如快速选择算法（寻找第k小的元素）
