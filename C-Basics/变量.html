C++ 变量与变量

掌握 C++ 变量类型、定义与作用域，构建程序的基本数据存储

数据类型

C++ 中的变量需要明确的数据类型，这决定了变量可以存储的数据种类以及占用的内存空间。以下是几种基本数据类型：

布尔类型 (bool)

用于逻辑运算，只有两个可能的值：

  • true - 真（非零）
  • false - 假（零）
bool is_correct = true;
bool has_error = false;

整数类型 (int)

用于存储整数，有多种大小：

  • short int - 短整型
  • int - 标准整型
  • long int - 长整型
  • long long int - 长长整型
int count = 42;
long long big_number = 9223372036854775807LL;

字符类型 (char)

用于存储单个字符，如字母、数字和符号：

  • 存储 ASCII 码表中的字符
  • 通常占用 1 个字节
  • 字符常量用单引号 'a' 表示
char letter = 'A';
char digit = '5';
char symbol = '+';
// 也可以用 ASCII 码值
char newline = 10; // 等同于 '\n'

浮点类型

用于存储带小数点的数值：

  • float - 单精度浮点数
  • double - 双精度浮点数
  • long double - 扩展精度浮点数
float price = 9.99f;
double pi = 3.14159265358979;
long double precise = 3.14159265358979323846L;

定宽整数类型

C++11 引入了定宽整数类型，可以确保整数类型在不同平台上具有相同的位宽：

#include <cstdint> // 需要包含此头文件

// 有符号定宽整数类型
int8_t a = -128;       // 8位有符号整数
int16_t b = 32767;     // 16位有符号整数
int32_t c = 2147483647;// 32位有符号整数
int64_t d = 9223372036854775807; // 64位有符号整数

// 无符号定宽整数类型
uint8_t e = 255;       // 8位无符号整数
uint16_t f = 65535;    // 16位无符号整数
uint32_t g = 4294967295; // 32位无符号整数
uint64_t h = 18446744073709551615U; // 64位无符号整数

定宽整数的优势

定宽整数类型可以确保在不同系统和编译器下变量的大小保持一致，有助于编写可移植的代码，特别是处理二进制数据或需要确切控制内存布局时非常有用。

类型转换

C++ 中数据类型之间可以进行转换：

隐式转换

编译器自动执行的类型转换：

int i = 5;
double d = i;    // int 隐式转换为 double，d = 5.0
char c = 65;     // int 隐式转换为 char，c = 'A'
int b = true;    // bool 隐式转换为 int，b = 1

显式转换（强制类型转换）

程序员明确指定的类型转换：

double pi = 3.14159;
int rounded = (int)pi;        // C风格强制转换，rounded = 3
int rounded2 = static_cast<int>(pi); // C++风格转换，rounded2 = 3

char c = 'A';
int ascii = static_cast<int>(c); // ascii = 65

类型转换可能导致数据精度丢失（如浮点数转整数时小数部分被截断）或者数据溢出（如较大的整数转换为较小的整数类型）。进行类型转换时应当注意可能的风险。

定义变量

定义变量需要指定变量类型和变量名。变量名是标识符，必须遵循C++命名规则。

// 基本变量定义语法
int number;         // 定义整型变量，未初始化
double price = 9.99; // 定义浮点型变量并初始化
char grade = 'A';   // 定义字符型变量并初始化

变量定义的注意事项：

  • 全局变量（定义在函数外）会被自动初始化为0
  • 局部变量（定义在函数内）不会自动初始化，使用前必须赋值
  • 变量名区分大小写，命名应有意义以提高代码可读性

变量作用域

变量的作用域决定了变量在程序中的可见性和生命周期。C++中主要有两种作用域：

全局作用域

全局变量定义在所有函数外部，程序从变量定义点到文件结束处都可以访问。

int global_var = 100; // 全局变量

void function() {
    // 可以访问 global_var
    global_var = 200;
}

局部作用域

局部变量定义在函数或代码块内部，只能在定义它的块内部访问。

void function() {
    int local_var = 50; // 局部变量
    
    {
        int block_var = 25; // 块作用域变量
        // 可以访问 local_var 和 block_var
    }
    // 这里不能访问 block_var
}
// 变量作用域示例
#include <iostream>
using namespace std;

int g = 20;  // 定义全局变量

int main() {
    int g = 10;         // 定义局部变量
    cout << g << endl;  // 输出 10（局部变量）
    
    {
        int g = 5;      // 在内部块中再次定义同名变量
        cout << g << endl;  // 输出 5（最内层的局部变量）
    }
    
    cout << g << endl;  // 输出 10（外层局部变量）
    cout << ::g << endl; // 输出 20（使用作用域解析运算符访问全局变量）
    
    return 0;
}

变量遮蔽警告

当在内层作用域定义了与外层作用域同名的变量时，内层变量会"遮蔽"外层变量。在代码块内，只能访问到最内层作用域中的同名变量。这种情况可能导致意外的错误，应尽量避免变量名重复的情况。

常量

常量是在程序执行期间不会改变的固定值。通过 const 关键字定义常量可以增加代码的安全性。

// 定义常量
const double PI = 3.14159265358979323846;
const int MAX_STUDENTS = 30;

// 尝试修改常量会导致编译错误
// PI = 3.14; // 错误：不能修改常量

常量的优势

  • 防止程序意外修改数值
  • 提高代码可读性，标记特殊数值
  • 便于维护，只需修改一处定义
  • 编译器可以进行更多优化

常见常量用法 

// 数学常量
const double PI = 3.14159265358979323846;
const double E = 2.71828182845904523536;

// 系统限制常量
const int MAX_USERS = 1000;
const int BUFFER_SIZE = 4096;

// 配置常量
const char* SERVER_IP = "192.168.1.1";
const int PORT = 8080;
