C++ 语法基础

掌握 C++ 编程的核心语法知识，为算法编程打下坚实基础

代码框架

如果你不想深究背后的原理，初学时可以直接将这个「框架」背下来：

#include <iostream>

int main() {
    // 这里是主函数内部
    
    return 0;
}

OI 中，程序的目的是完成题目所要求的任务，接受输入，进行计算，之后输出结果。源代码的基本内容是引入必须的库（头文件）、定义主函数、读入数据、计算、输出，最后结束。

注释

注释是程序中用于解释和说明的文字，编译器会忽略这些内容。C++ 支持两种注释方式：

// 这是单行注释

/*
这是多行注释
可以跨越多行
*/

#include <iostream>

int main() {
    // 这是程序的主函数
    std::cout << "Hello, World!" << std::endl; // 输出 Hello, World!
    return 0; // 返回 0 表示程序正常结束
}

良好的注释习惯

  • 在代码的关键部分添加注释，解释算法思路
  • 注释应当简洁明了，避免过多的废话
  • 对于复杂的函数，在函数开头添加功能说明
  • 在大型项目中，保持注释风格的一致性

输入与输出

C++ 提供了多种输入输出方式，以下介绍两种常用的方法：

cin 与 cout

C++ 标准库提供的输入输出流对象，需要包含 <iostream> 头文件：

#include <iostream>

int main() {
    int a, b;
    std::cin >> a >> b;         // 从标准输入读取两个整数
    std::cout << a + b << std::endl;  // 将两个整数的和输出到标准输出
    return 0;
}

也可以使用 using namespace std; 来简化代码，但这在较大的项目中可能导致命名冲突

scanf 与 printf

C 语言的输入输出函数，C++ 也可以使用它们，通常比 cin 和 cout 执行速度更快：

#include <cstdio>  // 或 #include <stdio.h>

int main() {
    int a, b;
    scanf("%d%d", &a, &b);       // 从标准输入读取两个整数
    printf("%d\n", a + b);       // 将两个整数的和输出到标准输出
    return 0;
}

格式说明符：%d（整数）、%f（浮点数）、%c（字符）、%s（字符串）、%lld（长整数）

性能提示

在 OI 竞赛中，使用 scanf/printf 通常比 cin/cout 更高效。如果必须使用 cin/cout，可以使用以下代码来加速： 

std::ios::sync_with_stdio(false);
std::cin.tie(nullptr);

但是，使用这段代码后就不能再使用 scanf/printf 了。

#define 命令

#define 是一种预处理命令，用于定义宏，本质上是文本替换：

#include <iostream>
#define n 233  // n 不是变量，而是编译器会将代码中所有 n 文本替换为 233

int main() {
    std::cout << n;  // 输出 233
    return 0;
}

需要注意的是，作为标识符一部分的 n 不会被替换，如 fn 不会被替换成 f233，同样，字符串内的也不会被替换。

什么是标识符？

标识符就是可以用作变量名的一组字符。例如，abcd 和 abc1 都是合法的标识符，而 1a 和 c+b 都不是合法的标识符。

标识符由英文字母、下划线开头，中间只允许出现英文字母、下划线和数字。值得注意的是，关键字（如 int, for, if）不能用作标识符。

宏可以带参数，带参数的宏可以像函数一样使用：

#include <iostream>
#define sum(x, y) ((x) + (y))
#define square(x) ((x) * (x))

int main() {
    std::cout << sum(1, 2) << ' ' << 2 * sum(3, 5) << std::endl;  // 输出 3 16
    return 0;
}

警告

带参数的宏和函数有区别。因为宏是文本替换，所以会引发许多问题： 

#include <iostream>
#define sum(x, y) x + y  // 这里应当为 #define sum(x, y) ((x) + (y))
#define square(x) ((x) * (x))

int main() {
    std::cout << sum(1, 2) << ' ' << 2 * sum(3, 5) << std::endl;
    // 输出为 3 11，因为 #define 是文本替换，后面的语句被替换为了 2 * 3 + 5
    
    int i = 1;
    std::cout << square(++i) << ' ' << i;
    // 输出未定义，因为 ++i 被执行了两遍
    // 而同一个语句中多次修改同一个变量是未定义行为（有例外）
    return 0;
}

使用 #define 是有风险的（由于 #define 作用域是整个程序，因此可能导致文本被意外地替换，需要使用 #undef 及时取消定义），因此应谨慎使用。较为推荐的做法是：使用 const 限定符声明常量，使用函数代替宏。

OI 中的常见用法

  • #define int long long + signed main()
    避免忘记开 long long 导致的错误
  • #define For(i, l, r) for (int i = (l); i <= (r); ++i)
    用于减短代码长度
  • #define pb push_back
    简化常用操作
  • #define mid ((l + r) / 2)
    简化二分搜索等算法

以上用法虽然在 OI 中常见，但不被推荐，会降低代码的规范性

条件编译

#ifdef LINUX
// code for linux
#else
// code for other OS
#endif

可以在编译的时候通过 -DLINUX 来控制编译出的代码，而无需修改源文件。

优点：通过 -DLINUX 编译出的可执行文件里并没有其他操作系统的代码，那些代码在预处理的时候就已经被删除了。
